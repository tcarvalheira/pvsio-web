Module: Wave

Renders a wave shape This module provides an API for setting up visual appearence, e.g., wave or background color
Version:
  • 1.0
Author:
  • Tiago Carvalheira
Example

Typical use of Wave APIs within a PVSio-web plugin module.

// Example module that uses Syringe.
define(function (require, exports, module) {
    "use strict";
    var device = {};
    device.syringe = new Syringe("salive", { top: 222, left: 96, height: 8, width: 38 });
    device.syringe.render();
});

Example

Typical use of Wave APIs within a PVSio-web plugin module.

// Example module that uses Syringe.
define(function (require, exports, module) {
    "use strict";
    var device = {};
    device.wave = new Wave("ecg", { top: 222, left: 96, height: 8, width: 38 }, { waveType: 'pleth', 
                                                        parent: 'prototype',
                                                        heartRate: 60,
                                                        waveColor: '#FFFFFF',
                                                        background: '#000000',
                                                        scanBarWidth: 10,
                                                        wavesPerScreen: 4.5
                                                    });
    device.wave.render();
});

(protected) addArrayElems()

This function defines the points of wave in order to get the suitable size

(protected) animationLoop()

This function is responsible for render wave. It is recursive so that the function is a render loop. In majoraty of Browsers the requestAnimationFrame is called at 60Hz so it is assumed that at speed 1, as we set process 2 pixel per function call, a wave needs to have 120 points to be rendered at 60bps

Name Type Description
id String The ID of the display.
coords Object The four coordinates (top, left, width, height) of the syringe, specifying the left, top corner, and the width and height of the div element containing the syringe. Default is { top: 0, left: 0, width: 80, height: 200 }.
opt Object Options:
Properties
Name Type Attributes Default Description
parent String <optional>
 'body' (String): the HTML element where the display will be appended (default is "body")
title String <optional>
 presented on top left corner with wave's color
heartRate Integer <optional>
 60 Heart rate in BPM. This will affect the speed of wave. Default is 60
waveColor uInt <optional>
 '#00FF00' (uInt). Default is #00FF00
backgroundColor uInt <optional>
 '#000000' widget background color. Default is #000
waveType 'ecg' | 'co2' | 'pleth' | 'userDefined' <optional>
 'ecg' Define the wave type that will be rendered. options are: ecg, co2, pleth, userDefined. Default is ecg
userDefinedWave Array.<Integer> (Array[Integer]) [optional] when waveType is userDefined the wave rendered will be the points in this array
filled 'none' | 'up' | 'down' <optional>
 'none' (string): Default is none
fillColor uInt <optional>
 waveColor (uInt). Default is wavecolor
wavesPerScreen Integer <optional>
 set the number of waves that should be rendered in each line of the screen at 60bps
scanBarWidth Integer <optional>
 50 set scanBarWidth, Default is 50
pvsDefinition Object <optional>
 defines the pvs state variables that holds waves definition

    color (uInt) rectagle color. Default is #000
    duration (Integer) time in ms that rectangle will take to go through all svg width. Usually this duration is a function of heart rate and number of beats that exists on svg. Default is 1000
